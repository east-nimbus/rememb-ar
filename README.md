# rememb-ar
RUNTEQ卒業制作

## サービス概要
バーで楽しんだカクテルを「自分だけの図鑑」としてコレクションし、知識と体験を積み上げていく記録アプリです。
写真・カクテル名・(ツイストであれば)ベース・店名をシンプルに記録し、記憶から溢れがちな「あの一杯」を確実にストックします。
自分の飲酒履歴を可視化することで、過去に飲んだ類似のカクテルを思い出して比較したり、自分のよく飲むカクテルの傾向を見つけることをお手伝いします。

## このサービスへの思い・作りたい理由

私はバーでお酒を楽しむ時間が好きですが、素敵なカクテルに出会って写真を撮っても、後で見返した時に「ベースは何だったか」「どこの店だったか」を思い出せず、知識として定着しないことに悔しさを感じていました。
バーの落ち着いた雰囲気を壊さず、かつ「ポケモン図鑑」のように自分が体験したお酒が一つずつ埋まっていく高揚感があれば、バー巡りがもっと知的で楽しい「探索」に変わるはずだ、という思いからこのサービスを構想しました。

## ユーザー層について

* **対象とするユーザー層** :
    * **バー初心者〜中級者：** カクテルの名前を覚えたいが、目の前でメモを取るのは気恥ずかしい、あるいは雰囲気を損なうと感じている方。スマホでカクテルの写真を撮ったのに、後からあまり見返さずにいる方。
    * **自分の好みを把握したい方：** 自分が「ジンのカクテルが好き」「度数が低いものが好き」といった傾向を客観的に知りたい方。
    * **ポケモンが好きな方:** ポケモン図鑑を使ってみたいと一度でも思ったことのある方。

* **選定理由：** 
    * 私自身が最大のターゲットであり、お酒の知識が定着することでバーテンダーさんとの会話の質が向上する喜びを、同じ悩みを持つユーザーに届けたいため。

## サービスの利用イメージ

1. **記録：** バーでカクテルが提供された際、写真を1枚撮影。カクテルを待つ間や飲んでいる最中の数秒で、カクテル名・店名・ベース・自分の好みだったかを選択して登録します。余裕があれば、選択情報に加えて、カクテルを作ってくれた人などの詳細情報も入力します。
2. **振り返り：** 
    * **注文前の振り返り**: 次に別のバーを訪れた際、図鑑を開いて「前回飲んだジンベースのものは少し強かったから、今回は同じジンベースでも度数の低めの甘いものを」といった、過去の経験に基づいた具体的な注文が可能になります。
    * **飲んだカクテルの傾向を分析**: 自分が良く飲むカクテルや、好みの度合いの記録に基づいて、飲んだカクテルの傾向を分析します。
    * **説明の自動生成**: 自分で細かい説明を書くのが面倒なら、キーワードだけ記録して、AIに自動で説明文を考えてもらうのも面白いでしょう。ポケモン図鑑のように「勝手に説明が埋まっていく」という体験は楽しいはずです。

3. **価値：** 単なるログではなく、使えば使うほど自分の好みの「傾向」が蓄積され、バーでの注文の失敗が減り、より自分に合った一杯に出会えるようになります。また傾向の分析がたまっていけば、それに合った提案をアプリからできるでしょうし、AIに自動で説明文を生成させるという体験が実現すれば、カクテル版ポケモン図鑑を手に入れたような面白さを体験してもらえるはずです。

## ユーザーの獲得について

* **身近な人から直接:** MVPレベルの最小限の機能しかないうちは、行きつけのバーなどで自分の作ったものとして直接紹介する形でユーザーを獲得し、アプリの洗練を図っていきます。
* **SNS（X/Instagram）での発信：** 機能が充実して安定した楽しさを実現できるようになってきたら、バー巡りを楽しむ層に向けて、カクテル図鑑がコレクションされていく視覚的な楽しみを発信していきます。

## サービスの差別化ポイント・推しポイント
### 差別化ポイント
1. **雑音のない「自分とお酒の対話空間」を提供**
InstagramやXなど既存のSNSは、他人の投稿や広告といった「外部のノイズ」が非常に多いメディアです。rememb-arはカクテルの記録だけに特化することで、他の情報が入る隙を与えません。これにより、過去に自分が飲んだ一杯と向き合い、「あの時の味」や「その時の感情」を純粋に振り返る、パーソナルな対話空間を提供します。

2. **「体験」を「知識」に変えるAI自動生成機能**
既存のSNSは、ユーザーが自分で文章を考えて投稿する必要がありますが、バーという暗くリラックスした空間ではその作業が負担になることもあります。rememb-arは、写真を撮るという最小限のアクションに対し、生成AIがそのカクテルのスタンダードな由来や特徴を「図鑑説明」として自動補完します。 「自分の体験」と「AIによる文章生成」が結びつくことで、ポケモン図鑑を作っていくようなハイテクな感動と、確実な知識の定着を両立させます。

3. **「未踏領域」の可視化で、未知の一杯へ導く契機を提供**
単なるログ（過去の記録）で終わるSNSと異なり、ベース酒別のカテゴリ表示によって「自分がまだ飲んでいないジャンル」を視覚的に浮き彫りにします。これにより、次にバーを訪れた際の注文が「いつもの一杯」から「図鑑を埋めるための新しい挑戦」へと変わり、ユーザーの行動変容を促します。

### 推しポイント
* **「図鑑」インターフェース：** 一般的なSNS（時系列表示）とは異なり、ベース酒（ジン・ラム等）や度数別のカテゴリ表示に特化。自分の好みの偏りや「まだ飲んでいない系統」がひと目で分かり、コンプリート意欲を刺激します。
* **入力のシンプルさ：** 多機能さをあえて削ぎ落とし、「バーのカウンターでサッと使える」ことを最優先にしたUI/UXを追求しています。
* **新しい出会いへの導き：** まだ飲んでいないジャンルを可視化することで、新しいカクテルとの出会いのモチベーションを創出します。

## 機能候補
### MVPリリース時（最小限のサービス検証）

* **ユーザー登録・ログイン機能：** 自分だけの非公開図鑑を保持するために最低限実装。
* **カクテル投稿機能：** 
    - 必須項目:
        - 写真
        - カクテル名
        - ベース（選択式）
        - アルコール度数（選択式）
    - オプション項目:
        - 店名
        - 好み度（5段階）
        - メモ
* **マイ図鑑表示・検索機能：** 
    - 登録したカクテル一覧を「ベース別」と「アルコール度数別」に表示。
* **検証の核：** 「手動でサッと記録し、ベース別やアルコール度数別に振り返る」ことが、次の注文の質を上げるかを確認します。

### 本リリース時

* **Google Maps連携：** 訪問したバーを地図上で可視化。
* **スタンダードカクテルコンプリート図鑑：** 登録したカクテルが特定のスタンダードカクテルであるか、そのツイストであれば、スタンダードカクテルコンプリート図鑑に「スタンダードカクテル」か「ツイスト」として登録される。
* **図鑑コンプリート進捗表示：** スタンダードカクテルコンプリート図鑑の進捗状況をパーセンテージで表示。
* **AI説明文生成：** メモやアルコール度数、ベースなどの情報から、自分だけのカクテル図鑑の説明文を自動生成する。

## 使用する技術スタック

### 実装イメージ

* **画像管理：** `Active Storage`を使用し、`Cloudinary`等の外部ストレージと連携して写真を管理。
* **UI構築：** `Tailwind CSS`と`daisyUI`を使用。バーの雰囲気に合うダークトーンのテーマを採用します。

### 技術構成

* **フレームワーク：** Ruby on Rails 7.x (Ruby 3.2.x)
* **データベース：** PostgreSQL
* **デプロイ先：** Heroku または Render
